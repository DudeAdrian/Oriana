# Docker Networking Guide for TikTok Clone\n\n## Network Architecture\n\n### Tiktok-Net Bridge Network\n- Subnet: 172.25.0.0/16\n- Gateway: 172.25.0.1\n- Isolated from host network by default\n\n### Service IPs (in tiktok-net)\n```\n172.25.0.2    postgres\n172.25.0.3    redis\n172.25.0.4    ipfs\n172.25.0.5    minio\n172.25.0.6    server\n172.25.0.7    nginx\n172.25.0.8    prometheus\n172.25.0.9    elasticsearch\n```\n\n## Service Communication\n\n### Internal (Container-to-Container)\n- PostgreSQL: postgres:5432\n- Redis: redis:6379\n- IPFS API: ipfs:5001\n- IPFS Gateway: ipfs:8080\n- MinIO: minio:9000\n- Server (API): server:5000\n- Nginx (Gateway): nginx:80/443\n- Prometheus: prometheus:9090\n- Elasticsearch: elasticsearch:9200\n\n### External (Host-to-Container)\n```\nPostgres:       localhost:5432\nRedis:          localhost:6379\nIPFS API:       localhost:5001\nIPFS Gateway:   localhost:8080\nMinIO API:      localhost:9000\nMinIO Console:  localhost:9001\nServer API:     localhost:5000\nNginx HTTP:     localhost:80\nNginx HTTPS:    localhost:443\nPrometheus:     localhost:9090\nElasticsearch:  localhost:9200\n```\n\n## Network Security\n\n### Port Exposure Rules\n- ✓ Public: 80, 443 (HTTP/HTTPS via Nginx)\n- ✓ Public: 5000 (API direct, optional)\n- ✓ Local: 5432, 6379, 9000, 9001 (Database, Cache, Storage)\n- ✗ Internal: 5001, 8080, 9090 (IPFS, Prometheus - only from containers)\n\n### Security Best Practices\n1. Only expose Nginx to the internet (ports 80, 443)\n2. Use environment variables for credentials\n3. Enable TLS/SSL for all external connections\n4. Implement rate limiting in Nginx\n5. Use Docker secrets for sensitive data in swarm mode\n\n## DNS Resolution\n\nInside containers, use service names as hostnames:\n```bash\n# From server container\ncurl http://postgres:5432  # PostgreSQL\nredis-cli -h redis        # Redis\ncurl http://minio:9000    # MinIO\n```\n\nFrom host machine:\n```bash\ncurl http://localhost:5000  # API\ncurl http://localhost:9001  # MinIO Console\n```\n\n## Custom Networks (Advanced)\n\nTo create additional isolated networks:\n\n```bash\ndocker network create tiktok-frontend\ndocker network create tiktok-backend\ndocker network create tiktok-storage\n```\n\nThen connect services to specific networks using docker-compose.\n
