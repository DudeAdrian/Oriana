# Phase 1 - Comprehensive Viewing and Demonstration Guide\n\n## 🎬 Overview\n\nThis guide will help you visualize and understand all components of your containerized TikTok Clone during Phase 1.  \n\n---\n\n## 🚀 Quick Start (5 minutes)\n\n### Step 1: Initialize Everything\n\n```bash\nchmod +x phase1-setup.sh\n./phase1-setup.sh\n```\n\nThis will:\n- Start all 8 containers\n- Wait for services to be healthy\n- Run database migrations\n- Create sample data\n- Display access information\n\n### Step 2: View Live Dashboard\n\n```bash\nchmod +x phase1-dashboard.sh\n./phase1-dashboard.sh\n```\n\nThis shows (updates every 5 seconds):\n- All service status (online/offline)\n- Container resource usage (CPU, Memory)\n- Access URLs and credentials\n- Quick commands\n- Troubleshooting tips\n\n### Step 3: Test All APIs\n\n```bash\nchmod +x phase1-test-api.sh\n./phase1-test-api.sh\n```\n\nThis tests:\n- User registration\n- User login\n- Profile retrieval\n- Video listing\n- Feed generation\n- Database connection\n- All storage services\n\n---\n\n## 📊 Service Access Points\n\n### 1. **API Server** (Port 5000)\n\n**URL**: http://localhost:5000  \n**Health Check**: http://localhost:5000/health\n\n**What to see**:\n```bash\n# Check API is running\ncurl http://localhost:5000/health\n\n# Response should be:\n{\"status\":\"ok\",\"message\":\"Server is running\"}\n```\n\n**View logs**:\n```bash\n./container-tools/manage.sh logs -f server\n```\n\n---\n\n### 2. **MinIO Console** (Port 9001) - S3 Storage\n\n**URL**: http://localhost:9001  \n**Username**: minioadmin  \n**Password**: minioadmin123\n\n**What to see**:\n1. Login page\n2. Dashboard showing storage buckets\n3. Object browser\n4. Access logs\n\n**Create a test bucket**:\n```bash\naws s3api create-bucket \\\n  --bucket test-bucket \\\n  --endpoint-url http://localhost:9000 \\\n  --create-bucket-configuration LocationConstraint=us-east-1\n```\n\n---\n\n### 3. **Prometheus Dashboard** (Port 9090) - Metrics\n\n**URL**: http://localhost:9090\n\n**What to see**:\n1. Main dashboard\n2. Graph tab for queries\n3. Alerts tab\n4. Status page\n\n**Common queries**:\n```\n# CPU usage\nprocess_cpu_system_seconds_total\n\n# Memory usage\nprocess_resident_memory_bytes\n\n# HTTP requests\nhttp_requests_total\n\n# Request duration\nhttp_request_duration_seconds\n```\n\n**How to query**:\n1. Click \"Graph\" tab\n2. Paste query above in expression field\n3. Click \"Execute\"\n4. See results in table or graph\n\n---\n\n### 4. **IPFS Web UI** (Port 5001) - Decentralized Storage\n\n**URL**: http://localhost:5001/webui\n\n**What to see**:\n1. IPFS node information\n2. Repository statistics\n3. Files browser\n4. Peers connected\n5. Stats and diagnostics\n\n**Upload a test file**:\n```bash\n# Create test file\necho \"Hello IPFS\" > test.txt\n\n# Upload to IPFS\ncurl -F file=@test.txt http://localhost:5001/api/v0/add\n\n# Response includes hash (hash value needed for retrieval)\n```\n\n---\n\n### 5. **PostgreSQL Database** (Port 5432)\n\n**Connection Details**:\n- Host: localhost:5432\n- Database: tiktok-clone\n- Username: user\n- Password: password\n\n**Connect and explore**:\n```bash\n# Via container tools\n./container-tools/manage.sh shell postgres\n\n# Or directly\npsql -h localhost -U user -d tiktok-clone\n\n# Common commands\n\\dt              # List tables\n\\d users         # Describe users table\nSELECT * FROM users;  # View users\nSELECT COUNT(*) FROM videos;  # Count videos\n```\n\n---\n\n### 6. **Redis Cache** (Port 6379)\n\n**Connection Details**:\n- Host: localhost:6379\n- Password: redispass\n\n**Connect and explore**:\n```bash\n# Via container tools\n./container-tools/manage.sh shell redis\n\n# Or directly\nredis-cli -h localhost -a redispass\n\n# Common commands\nPING                 # Test connection\nKEYS *              # List all keys\nGET <key>           # Get value\nTTL <key>           # Check expiration\nINFO memory         # Memory stats\n```\n\n---\n\n### 7. **Elasticsearch** (Port 9200) - Logs\n\n**URL**: http://localhost:9200\n\n**View cluster health**:\n```bash\ncurl http://localhost:9200/_cluster/health | jq .\n```\n\n**View indices**:\n```bash\ncurl http://localhost:9200/_cat/indices | head\n```\n\n**Search logs**:\n```bash\ncurl 'http://localhost:9200/_search?q=error' | jq .\n```\n\n---\n\n## 🧪 Testing Workflows\n\n### Workflow 1: Create New User and Login\n\n```bash\n# 1. Register new user\ncurl -X POST http://localhost:5000/api/auth/register \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"email\":\"player1@tiktok.app\",\n    \"username\":\"player1\",\n    \"password\":\"MyPassword123!\"\n  }'\n\n# 2. Login with that user\ncurl -X POST http://localhost:5000/api/auth/login \\\n  -H \"Content-Type: application/json\" \\\n  -d '{\n    \"email\":\"player1@tiktok.app\",\n    \"password\":\"MyPassword123!\"\n  }'\n\n# 3. Copy the token from response\n# 4. Use token for authenticated requests\ncurl -H \"Authorization: Bearer YOUR_TOKEN_HERE\" \\\n  http://localhost:5000/api/users/profile\n```\n\n---\n\n### Workflow 2: Explore Video Feed\n\n```bash\n# 1. Get trending videos\ncurl http://localhost:5000/api/feed/discover?limit=10\n\n# 2. Get personal feed (requires auth)\ncurl -H \"Authorization: Bearer YOUR_TOKEN\" \\\n  http://localhost:5000/api/feed?limit=10\n\n# 3. Search for users\ncurl 'http://localhost:5000/api/users/search?query=demo'\n\n# 4. Get specific user profile\ncurl http://localhost:5000/api/users/:userId\n```\n\n---\n\n### Workflow 3: Check Database State\n\n```bash\n# 1. Connect to database\n./container-tools/manage.sh shell postgres\n\n# 2. In PostgreSQL:\n\\c tiktok-clone\n\\dt              # Show tables\n\n# 3. View data\nSELECT * FROM users LIMIT 5;\nSELECT COUNT(*) FROM videos;\nSELECT * FROM notifications LIMIT 5;\n```\n\n---\n\n### Workflow 4: Monitor Performance\n\n```bash\n# Terminal 1: Monitor containers\n./container-tools/manage.sh stats\n\n# Terminal 2: Follow API logs\n./container-tools/manage.sh logs -f server\n\n# Terminal 3: Open Prometheus\n# http://localhost:9090\n# Query: rate(http_requests_total[1m])\n\n# Terminal 4: Run load test\n./phase1-load-test.sh\n```\n\n---\n\n## 📈 Key Metrics to Monitor\n\n### API Performance\n\n```bash\n# Check response times\ncurl -w \"Response time: %{time_total}s\\n\" -o /dev/null -s http://localhost:5000/api/videos\n\n# Output example:\n# Response time: 0.123456s\n```\n\n**Good response times**:\n- Health check: < 50ms\n- Simple queries: 100-200ms\n- Complex queries: 200-500ms\n- File uploads: varies with file size\n\n### Container Resources\n\n```bash\n# View resource usage\n./container-tools/manage.sh stats\n\n# Each service should use:\n# - CPU: < 30% each\n# - Memory: 100MB - 1GB depending on service\n```\n\n### Database Connections\n\n```bash\n# Check active connections\n./container-tools/manage.sh exec postgres psql -U user -d tiktok-clone -c \\\n  \"SELECT count(*) FROM pg_stat_activity;\"\n\n# Should be low (< 10) unless under heavy load\n```\n\n---\n\n## 🔍 Troubleshooting from Dashboard\n\n### Issue: API showing errors in logs\n\n**Solution**:\n```bash\n# 1. Check database connection\n./container-tools/manage.sh logs postgres | tail -20\n\n# 2. Verify migrations ran\n./container-tools/manage.sh exec server npx prisma migrate status\n\n# 3. Restart server\n./container-tools/manage.sh restart server\n```\n\n### Issue: High memory usage\n\n**Solution**:\n```bash\n# 1. Check which service\n./container-tools/manage.sh stats\n\n# 2. Check Redis memory\n./container-tools/manage.sh exec redis redis-cli info memory\n\n# 3. Clear old sessions/cache\n./container-tools/manage.sh exec redis redis-cli FLUSHALL\n```\n\n### Issue: Database insertion errors\n\n**Solution**:\n```bash\n# 1. Check schema\n./container-tools/manage.sh exec server npx prisma db push\n\n# 2. View table structure\n./container-tools/manage.sh shell postgres\n# Then: \\d tablename\n```\n\n---\n\n## 📋 Viewing Checklist\n\nBefore moving to Phase 2, verify:\n\n- [ ] `./phase1-setup.sh` completes without errors\n- [ ] All 8 services show ✓ ONLINE in dashboard\n- [ ] MinIO console accessible at http://localhost:9001\n- [ ] API health check returns 200 OK\n- [ ] `./phase1-test-api.sh` all tests pass\n- [ ] Database has tables and sample data\n- [ ] Prometheus showing metrics\n- [ ] Can create new user and login\n- [ ] Can view videos and feed\n- [ ] Container resource usage is reasonable\n- [ ] No errors in service logs\n\n---\n\n## 🎯 Next Phase Preparation\n\nOnce Phase 1 is verified:\n\n1. **Document Current State**\n   ```bash\n   ./container-tools/manage.sh backup\n   ```\n\n2. **Prepare AWS Credentials**\n   -
