# Docker Container Tools Guide\n\n## Overview\n\nComplete containerized environment for TikTok Clone with production-ready configurations.\n\n## Quick Start\n\n### Start All Services\n\n**Linux/Mac:**\n```bash\nchmod +x container-tools/manage.sh\n./container-tools/manage.sh up\n```\n\n**Windows (PowerShell):**\n```powershell\n.\\container-tools\\manage.cmd up\n```\n\n**Using Docker Compose directly:**\n```bash\ndocker-compose up -d\n```\n\n### Verify Services\n\n```bash\n# Check all containers\n./container-tools/manage.sh status\n\n# Run health checks\n./container-tools/manage.sh health-check\n\n# View logs\n./container-tools/manage.sh logs -f server\n```\n\n## Container Services\n\n### 1. PostgreSQL (Database)\n- **Container**: tiktok-postgres\n- **Port**: 5432\n- **Image**: postgres:15-alpine\n- **Credentials**: user/password (configure in .env)\n- **Features**: \n  - Automatic backups\n  - Data persistence via volume\n  - Health checks\n  - Audit logging\n\n**Access:**\n```bash\n# Via container tools\n./container-tools/manage.sh shell postgres\n\n# Via psql\npsql -h localhost -U user -d tiktok-clone\n\n# Via Docker\ndocker exec -it tiktok-postgres psql -U user -d tiktok-clone\n```\n\n### 2. Redis (Cache)\n- **Container**: tiktok-redis\n- **Port**: 6379\n- **Image**: redis:7-alpine\n- **Features**:\n  - Session management\n  - Real-time data caching\n  - Rate limiting\n  - Automatic persistence (AOF)\n  - Password authentication\n\n**Access:**\n```bash\n# Via container tools\n./container-tools/manage.sh shell redis\n\n# Via redis-cli\nredis-cli -h localhost\n\n# Check memory\nredis-cli info memory\n```\n\n### 3. IPFS (Decentralized Storage)\n- **Container**: tiktok-ipfs\n- **API Port**: 5001 (internal)\n- **Gateway Port**: 8080 (internal)\n- **Image**: ipfs/go-ipfs:v0.17.0\n- **Features**:\n  - Content-addressed storage\n  - Global distribution\n  - Peer-to-peer networking\n  - CAR file support\n\n**Access:**\n```bash\n# IPFS HTTP API\ncurl http://localhost:5001/api/v0/id\n\n# Upload file\ncurl -X POST -F \"file=@video.mp4\" http://localhost:5001/api/v0/add\n\n# Web UI\nopen http://localhost:5001/webui\n```\n\n### 4. MinIO (S3-Compatible Storage)\n- **Container**: tiktok-minio\n- **API Port**: 9000 (internal)\n- **Console Port**: 9001 (host: 9001)\n- **Image**: minio/minio:latest\n- **Features**:\n  - AWS S3 compatible API\n  - Web console\n  - Multi-part uploads\n  - Access policies\n\n**Access:**\n```bash\n# MinIO Console\nopen http://localhost:9001\n# Login: minioadmin / minioadmin123\n\n# AWS CLI\naws s3 ls --endpoint-url http://localhost:9000\n\n# Create bucket\naws s3api create-bucket --bucket tiktok-content --endpoint-url http://localhost:9000\n```\n\n### 5. Node.js Server (API)\n- **Container**: tiktok-server\n- **Port**: 5000\n- **Image**: custom (built from apps/server/Dockerfile)\n- **Features**:\n  - Express.js API\n  - Prisma ORM\n  - JWT authentication\n  - Real-time capabilities\n  - Health checks\n\n**Access:**\n```bash\n# Check API health\ncurl http://localhost:5000/health\n\n# View logs\n./container-tools/manage.sh logs -f server\n\n# Execute command in container\n./container-tools/manage.sh exec server npm run test\n```\n\n### 6. Nginx (Reverse Proxy)\n- **Container**: tiktok-nginx\n- **HTTP Port**: 80 (host: 80)\n- **HTTPS Port**: 443 (host: 443)\n- **Image**: nginx:alpine\n- **Features**:\n  - Load balancing\n  - Rate limiting\n  - GZIP compression\n  - SSL/TLS termination\n  - Caching\n  - Security headers\n\n**Access:**\n```bash\n# Test configuration\ndocker exec tiktok-nginx nginx -t\n\n# View current config\ndocker exec tiktok-nginx cat /etc/nginx/nginx.conf\n\n# Reload configuration\ndocker exec tiktok-nginx nginx -s reload\n```\n\n### 7. Prometheus (Monitoring)\n- **Container**: tiktok-prometheus\n- **Port**: 9090 (host: 9090)\n- **Image**: prom/prometheus:latest\n- **Features**:\n  - Metrics collection\n  - Time-series database\n  - Alerting (optional)\n  - PromQL queries\n\n**Access:**\n```bash\n# Prometheus Dashboard\nopen http://localhost:9090\n\n# Query metrics\ncurl http://localhost:9090/api/v1/query?query=up\n```\n\n### 8. Elasticsearch (Logging)\n- **Container**: tiktok-elasticsearch\n- **Port**: 9200 (host: 9200)\n- **Image**: docker.elastic.co/elasticsearch/elasticsearch:8.5.0\n- **Features**:\n  - Full-text search\n  - Log aggregation\n  - Analytics\n  - JSON REST API\n\n**Access:**\n```bash\n# Health check\ncurl http://localhost:9200/_cluster/health\n\n# Query indices\ncurl http://localhost:9200/_cat/indices\n```\n\n## Container Management\n\n### Using Container Tools Script\n\n#### Linux/Mac (Bash)\n\n```bash\n# Start all services\n./container-tools/manage.sh up\n\n# Stop all services\n./container-tools/manage.sh down\n\n# Show running containers\n./container-tools/manage.sh ps\n\n# Show status and health\n./container-tools/manage.sh status\n\n# View logs (follow mode)\n./container-tools/manage.sh logs -f server\n\n# Build images\n./container-tools/manage.sh build\n\n# Rebuild without cache\n./container-tools/manage.sh rebuild\n\n# Open shell in container\n./container-tools/manage.sh shell postgres\ncontainer-tools/manage.sh shell redis\n./container-tools/manage.sh shell server\n\n# Execute command in running container\n./container-tools/manage.sh exec server npm run test\n./container-tools/manage.sh exec postgres psql -U user -d tiktok-clone -c \"SELECT COUNT(*) FROM users;\"\n\n# Restart specific service\n./container-tools/manage.sh restart server\n\n# Scale service\n./container-tools/manage.sh scale server=3\n\n# Backup database\n./container-tools/manage.sh backup\n\n# Restore from backup\n./container-tools/manage.sh restore backups/tiktok-db-20260415_120000.sql\n\n# Monitor dashboards\n./container-tools/manage.sh monitor\n\n# Show resource usage\n./container-tools/manage.sh stats\n\n# Run health checks\n./container-tools/manage.sh health-check\n\n# Clean up (remove containers and volumes)\n./container-tools/manage.sh clean\n\n# Prune unused resources\n./container-tools/manage.sh prune\n```\n\n#### Windows (Command Prompt)\n\n```cmd\n# Start all services\ncontainer-tools\\manage.cmd up\n\n# Stop all services\ncontainer-tools\\manage.cmd down\n\n# Show container list\ncontainer-tools\\manage.cmd ps\n\n# Show status\ncontainer-tools\\manage.cmd status\n\n# View logs\ncontainer-tools\\manage.cmd logs server\n\n# Build images\ncontainer-tools\\manage.cmd build\n\n# Open shell\ncontainer-tools\\manage.cmd shell postgres\n\n# Backup database\ncontainer-tools\\manage.cmd backup\n\n# Health check\ncontainer-tools\\manage.cmd health-check\n```\n\n### Using Docker Compose Directly\n\n```bash\n# Start services\ndocker-compose up -d\n\n# Stop services\ndocker-compose down\n\n# View container status\ndocker-compose ps\n\n# View logs\ndocker-compose logs -f server\n\n# Build images\ndocker-compose build\n\n# Execute command\ndocker-compose exec server npm run test\n\n# Scale service\ndocker-compose up -d --scale server=3\n\n# Remove containers and volumes\ndocker-compose down -v\n```\n\n## Environment Configuration\n\n### Development (.env)\n```bash\nNODE_ENV=development\nLOG_LEVEL=debug\nDATABASE_URL=postgresql://user:password@postgres:5432/tiktok-clone\n```\n\n### Production (.env.production)\n```bash\nNODE_ENV=production\nLOG_LEVEL=info\nDATABASE_URL=postgresql://user:secure-password@postgres:5432/tiktok-clone\n```\n\n### Using Production Configuration\n\n```bash\n# Create production compose file\ndocker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d\n\n# With environment file\ndocker-compose --env-file .env.production up -d\n```\n\n## Volume Management\n\n### Named Volumes\n\n```bash\n# List all volumes\ndocker volume ls\n\n# Inspect volume\ndocker volume inspect app_postgres_data\n\n# Remove unused volumes\ndocker volume prune\n\n# Backup volume\ndocker run --rm -v app_postgres_data:/data -v $(pwd):/backup alpine tar czf /backup/postgres-backup.tar.gz -C /data .\n\n# Restore volume\ndocker run --rm -v app_postgres_data:/data -v $(pwd):/backup alpine tar xzf /backup/postgres-backup.tar.gz -C /data\n```\n\n## Networking\n\n### View Network\n\n```bash\n# List networks\ndocker network ls\n\n# Inspect network\ndocker network inspect app_tiktok-net\n\n# Test connectivity\ndocker exec tiktok-server ping postgres\ndocker exec tiktok-server curl http://redis:6379\n```\n\n## Database Operations\n\n### Prisma Migrations\n\n```bash\n# Run migrations in container\ndocker exec tiktok-server npx prisma migrate deploy\n\n# Create new migration\ndocker exec tiktok-server npx prisma migrate dev --name add_new_table\n\n# Open Prisma Studio\ndocker exec -it tiktok-server npx prisma studio\n```\n\n### Database Backup & Restore\n\n```bash\n# Backup using container tools\n./container-tools/manage.sh backup\n\n# Manual backup\ndocker exec tiktok-postgres pg_dump -U user -d tiktok-clone > backup.sql\n\n# Restore\ndocker exec -i tiktok-postgres psql -U user -d tiktok-clone < backup.sql\n```\n\n## Troubleshooting\n\n### Services Won't Start\n\n```bash\n# Check service logs\ndocker-compose logs postgres\ndocker-compose logs redis\n\n# Verify ports are available\nlsof -i :5432  # PostgreSQL\nlsof -i :6379  # Redis\nlsof -i :9000  # MinIO\n\n# Force restart\ndocker-compose restart\n```\n\n### High Memory/CPU Usage\n\n```bash\n# Check resource usage\ndocker stats\n\n# Limit container resources\ndocker update --memory 512m --cpus 1.0 tiktok-server\n```\n\n### Network Issues\n\n```bash\n# Check container connectivity\ndocker exec tiktok-server ping postgres\n\n# Check DNS resolution\ndocker exec tiktok-server nslookup redis\n\n# Inspect network\ndocker network inspect tiktok_tiktok-net\n```\n\n### Database Connection Issues\n\n```bash\n# Check if PostgreSQL is running\ndocker exec tiktok-postgres pg_isready -U user\n\n# Test connection from server\ndocker exec tiktok-server psql -h postgres -U user -d tiktok-clone -c \"SELECT 1;\"\n```\n\n## Performance Already configured in nginx/nginx.conf\n   - Update server blocks for your domain\n\n4. **Limit container privileges**\n   ```yaml\n   security_opt:\n     - no-new-privileges:true\n   ```\n\n## Production Deployment\n\n### AWS ECS/Fargate\n\n```bash\n# Build and push to ECR\naws ecr get-login-password --region ap-southeast-2 | docker login --username AWS --password-stdin 123456789.dkr.ecr.ap-southeast-2.amazonaws.com\n\ndocker build -t tiktok-clone:latest apps/server/\ndocker tag tiktok-clone:latest 123456789.dkr.ecr.ap-southeast-2.amazonaws.com/tiktok-clone:latest\ndocker push 123456789.dkr.ecr.ap-southeast-2.amazonaws.com/tiktok-clone:latest\n```\n\n### Docker Swarm\n\n```bash\n# Initialize swarm\ndocker swarm init\n\n# Create stack\ndocker stack deploy -c docker-compose.yml tiktok-clone\n\n# Check stack\ndocker stack ps tiktok-clone\n```\n\n## Further Resources\n\n- See [docker-network-guide.md](./docker-network-guide.md) for networking details\n- See [DEPLOYMENT_GUIDE.md](./DEPLOYMENT_GUIDE.md) for production deployment\n- See [docker-compose.yml](./docker-compose.yml) for service definitions\n- Docker Documentation: https://docs.docker.com\n
