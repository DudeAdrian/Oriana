Solution 2: Free up resources\ndocker-compose down\ndocker system prune -a\n\n# Solution 3: Check ports\nlsof -i :5000  # If ports in use\n\n# Solution 4: Rebuild\n./container-tools/manage.sh rebuild\n```\n\n### Problem: API not responding\n```bash\n# Check logs\n./container-tools/manage.sh logs -f server\n\n# Check database connection\n./container-tools/manage.sh logs postgres\n\n# Restart server\n./container-tools/manage.sh restart server\n```\n\n### Problem: Database issues\n```bash\n# Check PostgreSQL\n./container-tools/manage.sh logs postgres\n\n# Connect and verify\npsql -h localhost -U user -d tiktok-clone -c \"SELECT COUNT(*) FROM users;\"\n\n# Re-run migrations\n./container-tools/manage.sh exec server npx prisma migrate deploy\n```\n\n### Problem: High memory/CPU\n```bash\n# Check resource usage\n./container-tools/manage.sh stats\n\n# Restart problematic service\n./container-tools/manage.sh restart server  # or postgres, redis, elasticsearch\n\n# Clear cache\n./container-tools/manage.sh exec redis redis-cli FLUSHALL\n```\n\n---\n\n## 📚 Documentation Index\n\n| Document | Purpose | Read Time |\n|----------|---------|----------|\n|
